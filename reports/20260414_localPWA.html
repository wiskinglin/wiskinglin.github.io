Chapter 1

底層架構：大容量模型檔存儲對比

動輒數 GB 的 AI 模型對傳統存儲機制提出極端挑戰。以下為主流客戶端存儲方案優劣分析：

1

Cache API

適用: 靜態資源

設計用於 HTTP 請求快取。處理大容量模型時，整體存入會導致嚴重的磁碟膨脹 (Disk Bloat)，且每次讀取需將完整物件載入記憶體，極易觸發 V8 引擎 OOM 崩潰。

2

IndexedDB

適用: 結構化資料

基於 LevelDB/SQLite 的 NoSQL 資料庫。支援高達磁碟空間 80% 的配額，但處理數 GB 單一檔案時，其交易序列化與反序列化過程仍會產生不可忽視的效能開銷。

最佳實踐
3

OPFS

適用: 巨量二進制檔

Origin Private File System。提供位元組級 (Byte-by-Byte) 存取，支援就地讀寫與零序列化開銷。在 Web Worker 中調用同步方法，I/O 速度可達 IndexedDB 數倍。

💡

架構建議：雙層快取架構 (Two-tier Cache)

強烈建議以 IndexedDB 存儲輕量詮釋資料 (Metadata)、分詞器設定檔，並將龐大的二進制模型權重直接寫入 OPFS。確保極致載入速度與優雅降級機制。

Chapter 3

商業變現與資安防禦策略

邊緣推論改變了傳統 AI SaaS 成本結構，並引發針對白盒攻擊的新一代防禦挑戰。

💎 變現模式創新

  • 結果導向任務計費 (Outcome Pricing)

    放棄 Token 計費，依照用戶端「成功完成的業務動作」於背景扣款。

  • 雲端/邊緣混合定價 (Hybrid Tiering)

    前端草稿免費，判定複雜任務時無縫切換 70B 雲端模型並觸發計費。

  • 隱私合規保證溢價 (Privacy Premium)

    為金融/醫療販售「100% 離線合規認證」，賺取絕對不上雲的溢價。

  • 自攜算力折扣 (BYOD Discount)

    讓使用者的設備買單，同意調用終端 WebGPU 即給予授權費折扣。

🛡️

🛡️ 白盒逆向工程防禦

模型存於前端構成白盒攻擊環境，需導入終極防禦：

01. 白盒密碼學與混淆

將權重編碼至複雜查詢表，不進行明文矩陣乘法，使惡意方無法透過記憶體還原浮點數。

02. WASM 指令虛擬化

將演算法轉為自訂位元組碼並置於 WASM 內微型 VM 直譯。偵測到 DevTools 斷點立即觸發自毀污染記憶體。

03. 神經網路白盒浮水印

微調階段透過 Auxiliary Loss 植入專屬印記，即使遭實體竊取也能在法庭上提出無法篡改的鑑識鐵證。
