02 UNDER THE HOOD

V8 與 libuv 的雙重基石

V8 引擎： 負責同步解析與執行 JavaScript，核心包含機器碼編譯與 Heap 記憶體管理。

libuv 庫： 用 C 語言編寫，統括 Event Loop 與 Thread Pool。Event Loop 負責非同步任務循環，Thread Pool (預設 4 個) 則處理 DNS、File System 等阻塞操作。

EXECUTION MODEL

非同步委託機制

主執行緒永不等待 (Never Wait)。I/O 操作委託給核心或 Thread Pool，完成後回呼進入 Event Queue 由 Event Loop 送回 V8。

// CPU Intensive Task Fix in 2026

2026 EVOLUTION

ESM 終於打破高牆

Node.js v22+ 的劃時代更新：正式支援在 CommonJS 裡同步 require() ESM 模組，徹底終結多年遷移陣痛。Tree-shaking 成為現代伺服器部署標配，顯著優化記憶體佔用。

04. ARCHITECTURE PATTERNS

微服務 vs Serverless

Microservices (K8s)

  • 適合持續性流量
  • 強依賴 Circuit Breakers
  • Docker 化部署標準

Serverless (Lambda)

  • 適合突發事件驅動
  • 挑戰：Cold Start 痛點
  • 推薦輕量 Drizzle ORM

DEVOPS STRATEGY:

單機推 PM2 Cluster Mode，企業級推 Docker + Kubernetes (放棄 PM2 交給 K8s 原生調度)。

The 2026 Ecosystem Stack

Web Frameworks

NestJS (Enterprise)

強 DI 架構。推薦底層替換為 Fastify 適配器，兼顧結構與吞吐效率。

Fastify (Performance)

基於 Schema 驗證，提供極致 RPS。速度為 Express 的 2-3 倍。

Express (Speed)

最成熟生態，適合微型 API 或快速 MVP 疊代。

Persistence (ORM)

Prisma (DX King)

Auto-generated Types 提供防呆體驗，適合絕大多數 Greenfield 專案。

Drizzle (Serverless)

2026 黑馬。貼近原生 SQL 效能，極端輕量，冷啟動首選。

TypeORM (Legacy)

適合重度依賴物件導向 (Java/C# 轉型) 的大型傳統架構團隊。

HA & Communication
  • gRPC: 2026 微服務間通訊標準。強型別、HTTP/2 低延遲。
  • Kafka: 事件驅動架構 (EDA) 骨幹，解耦複雜流程。
  • Redis Cluster: 實現分散式鎖、限流與 Session 集群協同。
