# iTaiwan (ITW) Editor 產品願景與 UX 矩陣

> **文件版本**: v2.3 | **最後更新**: 2026-04-20
> **專案代號**: iTaiwan

## 一、 專案願景與核心理念

打造一種具備高度互操作性、人類可讀、且對 AI 極度友好的通用數據容器格式。透過系統底層的排版演算法與瀏覽器端的解析渲染，**將非結構化的「文字、數據、圖片」，全自動轉化為具備商業變現能力的高價值視覺 Web 應用程式**。

我們不再將其定位為傳統的通用文書軟體，而是「情境驅動的 Web App 引擎」。一個檔案就是一個專注解決特定問題的獨立應用站點（例如「預算規劃器」、「募資簡報工作站」或「商務合約編輯器」），直接賦能行銷、募資、合約、數據決策等商業場景，拒絕為無用的介面切換與過剩的選單買單。

資料不只是冷冰冰的數字，它應該自帶語意（Markdown）、結構（JSON）與呈現方式（HTML）。系統將運用強大的公式：**網頁最終呈現 = (內容語意 × 模組化設計) + (數據結構 × 排版演算法)**，提供極速的視覺體驗與雙向資料連動。

---

## 二、 破除傳統框架 (Deconstructing Traditional Suites)

為達成情境驅動的目標，我們汲取傳統工具的最強底層邏輯，並捨棄臃腫的黑箱設計：

* **Word 文本邏輯**: 擷取段落與樣式分離的概念，徹底拋棄封閉 OOXML，改以純 Markdown 作為文本操作核心。
* **Excel 算力繼承**: 核心採用純 CSV 儲存數據，並透過瀏覽器端 HyperFormula 引擎提供即時計算與圖表連動。
* **PowerPoint 敘事結構**: 借鑑區塊敘事思維（`---SLIDE:N---`），全面導入 CSS Flexbox/Grid 實現完美 RWD 佈局。
* **現代編輯器萃取**: 引進「高沉浸焦距寫作 (Zen Mode)」，將底層資料抽象化為 Block（概念類似 Notion），由前端自動決定排版。

---

## 三、 情境驅動的 Web App 設計

系統底層摒棄「通用三分頁介面 (MD/CSV/HTML 切換)」，轉向**「情境應用 Web App 生成器」**：

1. **資料本質宣告**: 透過檔案屬性標籤（如 `---THEME---`），系統精準判定資料容器的「應用情境」。
2. **動態專屬 UI 生成**: 前端不提供通用工具列，而是建構最符合該情境的專案介面。
3. **無縫雙向同步**: Block DOM 與純文字底層零延遲轉換。

---

## 四、 應用驅動 UI 矩陣 (Canvas 總覽)

為了最大化 `.itw` 高價值產出的目的，前端 UI 針對四大場景進行了「架構 -> 細節 -> 打磨」三部曲工作流的深度縫合：

### 4.1 簡報類 (Pitch Deck)：張力驅動的視覺敘事 (優先級 P0)
**代表場景：創作式敘事 (節奏控制) + 演繹式骨架 (MECE 說服力)。**
1. **商業提案：** 募資簡報 (Pitch Deck)、公司介紹 (Company Profile)、產品發表/路演 (Product Launch)、年度/季度業績報告。
2. **教育與學術：** 課堂教學投影片、學術研討會發表、學生專題報告、畢業答辯簡報。
3. **團隊與內部：** 員工培訓/入職簡報、行銷策略提案、腦力激盪與工作坊 (Workshop) 引導簡報。

**簡報產出流程：對場景與受眾進行 TA 設定後，透過架構、細節與視覺打磨三階段，打造充滿張力的敘事演說。**
1. **架構期 (構築敘事節拍)：** 使用者進入**「大綱投影片視圖 (Slide Outline View)」**。此階段系統隱藏畫布，強制使用者在左側面板拖曳節點。這並非單純排版，而是利用「節拍表 (Beat Sheet)」概念，編排從「痛點、解法到願景」的敘事曲線，確保邏輯的演繹嚴密性。
2. **細節期 (版型與內容填充)：** 架構底定後，展開**「---SLIDE:N--- 版型選擇器」**。系統根據大綱節點的語義屬性推薦單頁佈局，使用者在此專注填寫文字與替換佔位符。
3. **打磨期 (視覺焦點與情緒操控)：** 使用者啟用**「高亮數據標籤面板」**，針對核心數字設定自動放大或跳動特效；透過**「背景滿版 / 毛玻璃區塊開關」**創造商業提案需要的視覺張力。

### 4.2 圖片類 (Visual Content)：從碎片到精準打擊 (優先級 P0)
**代表場景：歸納式聚類 (原子化蒐集) + 創作式視覺 (引導點擊)。**
1. **社群媒體：** IG 貼文/限時動態、YT 影片封面 (Thumbnail)、FB 粉絲專頁封面、X (Twitter) 附圖。
2. **行銷與廣告：** 網站 Banner、電商商品圖 (商品上架圖)、Google 廣告素材、促銷活動 Pop-up 圖。
3. **活動與公關：** 活動海報、邀請卡片、生日卡片、節慶賀卡、宣傳傳單 (Flyer)、酷卡。
4. **品牌與個人：** 品牌 Logo、名片 (Business Card)、社群平台個人頭貼、手機/電腦桌布。

**圖片產出流程：從碎片化素材收集開始，透過科學化排版與無障礙檢測，精準打造高點擊率與高傳達率的視覺圖像。**
1. **架構期 (素材原子化與聚合)：** 起點在**「媒體資產庫面板 (Inbox)」**。將外部圖片、Icon 拖曳至此，轉為 Base64 本地端零延遲處理，歸納出情緒板 (Moodboard)。
2. **細節期 (科學化佈局)：** 預設啟動**「排版黃金三角對齊熱區顯示」**（基於眼動追蹤理論），引導核心轉化資訊（如行動呼籲）精準放置於高點擊熱區。
3. **打磨期 (無障礙與合規驗證)：** 透過**「智慧輔助配色檢測」**掃描暗黑模式或高風險區塊，確保符合 WCAG 高對比，避免視覺盲區。

### 4.3 試算表類 (DataGrid)：讓死數據開口說話 (優先級 P1)
**代表場景：演繹式 (MECE 分類) + 歸納式 (數據洞察與合成)。**
1. **專案與任務管理：** 甘特圖 (Gantt Chart)、專案進度追蹤表、待辦事項清單 (To-Do List)、員工排班表。
2. **財務與會計：** 年度預算規劃表、個人/家庭記帳本、公司損益表範本、差旅費用報銷單、投資回報 (ROI) 計算機。
3. **營運與行政：** 庫存盤點表、客戶聯絡資訊 (CRM 表格)、活動報名名冊追蹤、考勤紀錄表、商品訂單追蹤表。

**試算表產出流程：先確立嚴謹的資料維度，再進行高密度數據輸入，最後透過動態視覺化讓數據自動說故事。**
1. **架構期 (維度與邏輯定義)：** 預先定義資料欄位，確保符合 MECE 原則（不重疊無遺漏）。
2. **細節期 (高密度資料輸入)：** 進入**「網格式資料編輯器 (Direct DataGrid Editor)」**，剔除富文本干擾，專注矩陣填充。
3. **打磨期 (動態視覺化敘事)：** 透過**「即時圖表對應器 (Chart Mapper)」**將數據轉成視覺化圖表，配有**「滑鼠滾動播放」**與**「閾值觸發器 (Threshold Triggers)」**（滾動至關鍵節點時 UI 背景變暗/震動）來沉浸呈現冷硬數據。

### 4.4 文件類 (Document)：降噪與體驗收納 (優先級 P1)
**代表場景：演繹式結構 (Top-Down 分層)。**
1. **商務與職場：** 商業企劃書 (Proposal)、專案計畫書、會議記錄 (Meeting Minutes)、報價單 (Quote)、發票與收據、員工入職手冊、新聞稿 (Press Release)。
2. **求職與職涯：** 個人履歷表 (Resume/CV)、求職信 (Cover Letter)、作品集介紹、推薦信。
3. **教育與生活：** 讀書計畫、課堂講義範本、旅遊行程規劃表、日記/電子手帳、檢討報告。

**文件產出流程：建立清晰的樹狀目錄骨架後，藉助沉浸模式專注輸出，最終運用排版折疊與語意縮放提升閱讀體驗。**
1. **架構期 (樹狀目錄構建)：** 在大綱視圖定義 H1-H3 骨架，確定邏輯層次佈局。
2. **細節期 (無干擾約束輸出)：** 進入**「高沉浸寫作模式 (Zen Mode / WYSIWYG)」**。並可啟動**「約束模式 (Constraint Mode)」**強迫寫完當前子節字數方能跳段落。
3. **打磨期 (資訊層級收納與質感營造)：** 使用**「開關式自動收納 (Accordion)」**折疊冗長段落；透過**「語義縮放 (Semantic Zooming)」**讓讀者透過縮放觸控板漸進顯示內文；最後套用**「專業黑白高對比版型」**提升專業感。
