# iTaiwan (ITW) Editor 產品願景與 UX 矩陣

> **文件版本**: v2.2 | **最後更新**: 2026-04-20
> **專案代號**: iTaiwan

## 一、 專案願景與核心理念

打造一種具備高度互操作性、人類可讀、且對 AI 極度友好的通用數據容器格式。透過系統底層的排版演算法與瀏覽器端的解析渲染，**將非結構化的「文字、數據、圖片」，全自動轉化為具備商業變現能力的高價值視覺 Web 應用程式**。

我們不再將其定位為傳統的通用文書軟體，而是「情境驅動的 Web App 引擎」。一個檔案就是一個專注解決特定問題的獨立應用站點（例如「預算規劃器」、「募資簡報工作站」或「商務合約編輯器」），直接賦能行銷、募資、合約、數據決策等商業場景，拒絕為無用的介面切換與過剩的選單買單。

資料不只是冷冰冰的數字，它應該自帶語意（Markdown）、結構（JSON）與呈現方式（HTML）。系統將運用強大的公式：**網頁最終呈現 = (內容語意 × 模組化設計) + (數據結構 × 排版演算法)**，提供極速的視覺體驗與雙向資料連動。

---

## 二、 破除傳統框架 (Deconstructing Traditional Suites)

為達成情境驅動的目標，我們汲取傳統工具的最強底層邏輯，並捨棄臃腫的黑箱設計：

* **Word 文本邏輯**: 擷取段落與樣式分離的概念，徹底拋棄封閉 OOXML，改以純 Markdown 作為文本操作核心。
* **Excel 算力繼承**: 核心採用純 CSV 儲存數據，並透過瀏覽器端 HyperFormula 引擎提供即時計算與圖表連動。
* **PowerPoint 敘事結構**: 借鑑區塊敘事思維（`---SLIDE:N---`），全面導入 CSS Flexbox/Grid 實現完美 RWD 佈局。
* **現代編輯器萃取**: 引進「高沉浸焦距寫作 (Zen Mode)」，將底層資料抽象化為 Block（概念類似 Notion），由前端自動決定排版。

---

## 三、 情境驅動的 Web App 設計

系統底層摒棄「通用三分頁介面 (MD/CSV/HTML 切換)」，轉向**「情境應用 Web App 生成器」**：

1. **資料本質宣告**: 透過檔案屬性標籤（如 `---THEME---`），系統精準判定資料容器的「應用情境」。
2. **動態專屬 UI 生成**: 前端不提供通用工具列，而是建構最符合該情境的專案介面。
3. **無縫雙向同步**: Block DOM 與純文字底層零延遲轉換。

## 四、 應用驅動 UI 矩陣 (Canvas 總覽)

為了最大化 `.itw` 高價值產出的目的，前端 UI 針對四大場景進行了「架構 -> 細節 -> 打磨」三部曲工作流的深度縫合：

### 4.1 簡報類 (Pitch Deck)：張力驅動的視覺敘事 (優先級 P0)
**代表場景：** 投資募資提案、建築設計提案、餐廳營運分析。
**核心邏輯：** 創作式敘事 (節奏控制) + 演繹式骨架 (MECE 說服力)。

*   **架構期 (構築敘事節拍)：** 使用者進入**「大綱投影片視圖 (Slide Outline View)」**。此階段系統隱藏畫布，強制使用者在左側面板拖曳節點。這並非單純排版，而是利用「節拍表 (Beat Sheet)」概念，編排從「痛點、解法到願景」的敘事曲線，確保邏輯的演繹嚴密性。
*   **細節期 (版型與內容填充)：** 架構底定後，展開**「---SLIDE:N--- 版型選擇器」**。系統根據大綱節點的語義屬性推薦單頁佈局，使用者在此專注填寫文字與替換佔位符。
*   **打磨期 (視覺焦點與情緒操控)：** 使用者啟用**「高亮數據標籤面板」**，針對核心數字設定自動放大或跳動特效；透過**「背景滿版 / 毛玻璃區塊開關」**創造商業提案需要的視覺張力。

### 4.2 圖片類 (Visual Content)：從碎片到精準打擊 (優先級 P0)
**代表場景：** 社群行銷傳單、產品展示圖、YouTube 縮圖、品牌標誌。
**核心邏輯：** 歸納式聚類 (原子化蒐集) + 創作式視覺 (引導點擊)。

*   **架構期 (素材原子化與聚合)：** 起點在**「媒體資產庫面板 (Inbox)」**。將外部圖片、Icon 拖曳至此，轉為 Base64 本地端零延遲處理，歸納出情緒板 (Moodboard)。
*   **細節期 (科學化佈局)：** 預設啟動**「排版黃金三角對齊熱區顯示」**（基於眼動追蹤理論），引導核心轉化資訊（如行動呼籲）精準放置於高點擊熱區。
*   **打磨期 (無障礙與合規驗證)：** 透過**「智慧輔助配色檢測」**掃描暗黑模式或高風險區塊，確保符合 WCAG 高對比，避免視覺盲區。

### 4.3 試算表類 (DataGrid)：讓死數據開口說話 (優先級 P1)
**代表場景：** 企業年度財報、營銷成長數據、暗黑幣價追蹤表。
**核心邏輯：** 演繹式 (MECE 分類) + 歸納式 (數據洞察與合成)。

*   **架構期 (維度與邏輯定義)：** 預先定義資料欄位，確保符合 MECE 原則（不重疊無遺漏）。
*   **細節期 (高密度資料輸入)：** 進入**「網格式資料編輯器 (Direct DataGrid Editor)」**，剔除富文本干擾，專注矩陣填充。
*   **打磨期 (動態視覺化敘事)：** 透過**「即時圖表對應器 (Chart Mapper)」**將數據轉成視覺化圖表，配有**「滑鼠滾動播放」**與**「閾值觸發器 (Threshold Triggers)」**（滾動至關鍵節點時 UI 背景變暗/震動）來沉浸呈現冷硬數據。

### 4.4 文件類 (Document)：降噪與體驗收納 (優先級 P1)
**代表場景：** 商業合約文件、科技硬體規格書、豪宅建案手冊。
**核心邏輯：** 演繹式結構 (Top-Down 分層)。

*   **架構期 (樹狀目錄構建)：** 在大綱視圖定義 H1-H3 骨架，確定邏輯層次佈局。
*   **細節期 (無干擾約束輸出)：** 進入**「高沉浸寫作模式 (Zen Mode / WYSIWYG)」**。並可啟動**「約束模式 (Constraint Mode)」**強迫寫完當前子節字數方能跳段落。
*   **打磨期 (資訊層級收納與質感營造)：** 使用**「開關式自動收納 (Accordion)」**折疊冗長段落；透過**「語義縮放 (Semantic Zooming)」**讓讀者透過縮放觸控板漸進顯示內文；最後套用**「專業黑白高對比版型」**提升專業感。
