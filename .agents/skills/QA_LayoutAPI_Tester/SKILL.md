# Layout QA Tester — 自動化排版與 API 穩定度測試

## 概述

此技能賦予 QA Agent 兩大核心驗證能力：
1. **切版穩定度測試**：驗證長篇報告套用 50 種博物館 CSS 風格時的排版品質
2. **Web API 穩定度測試**：測試 File System Access API 等瀏覽器 API 在各種異常狀態下的防呆處理

## 系統狀態與會話交接 (Memory & Sessions)

- **前置讀取**：執行任務前，必須優先讀取 `.agents/memory/preferences.json` (獲取全域偏好) 與 `.agents/memory/lessons_learned.md` (規避已知錯誤)。
- **會話交接**：完成任務後，除輸出正式檔案外，須將執行狀態摘要寫入或更新當前的 Workflow Session 檔 (`.agents/sessions/session-{id}.md`) 供下游 Agent 閱讀。

## 必備專業技能

### 1. 切版驗證技術 (Layout Verification)

- **視覺回歸測試**：
  - 截圖比對（Pixel-level comparison）：偵測意外的視覺改動
  - 佈局斷言（Layout Assertions）：驗證元素位置、尺寸、間距
  - Overflow 偵測：檢查文字/圖片是否超出容器邊界
- **邊界條件測試**：

| 測試維度 | 極限場景 |
|---------|---------|
| 文字長度 | 萬字長文、單字超長（無空白斷行點）、空內容 |
| 圖表尺寸 | 巨大圖表（4000px 寬）、極小圖表（50px）、無圖 |
| 標題層級 | 連續 6 層嵌套標題、超長單行標題 |
| 表格規模 | 20 欄 × 100 列的密集表格 |
| 清單深度 | 5 層嵌套有序/無序清單 |
| 程式碼區塊 | 500 行超長程式碼、含特殊字元 |

- **響應式斷點驗證**：
  - 桌面（1440px / 1024px）
  - 平板（768px）
  - 手機（375px / 320px）

### 2. Web API 異常測試 (API Robustness Testing)

- **File System Access API 異常場景**：

| 場景 | 預期行為 |
|------|---------|
| 使用者拒絕授權 | 友善提示 + 降級為下載模式 |
| 磁碟空間不足 | 顯示空間不足警告 + 提供清理建議 |
| 檔案被其他程式鎖定 | 顯示鎖定提示 + 重試按鈕 |
| 存檔路徑不存在 | 自動建立目錄或提示選擇新路徑 |
| 瀏覽器不支援 API | Feature Detection + 降級為傳統下載 |
| 存檔過程中瀏覽器關閉 | Auto-save 機制確保資料不遺失 |

- **ContentEditable 異常場景**：

| 場景 | 預期行為 |
|------|---------|
| 貼上含惡意 HTML/JS | XSS 過濾 + 僅保留純文字或安全標籤 |
| 貼上超大量內容（10MB+） | 效能保護 + 漸進式載入 |
| 快速連續 Undo/Redo | 歷史紀錄穩定、不崩潰 |
| 同時多處編輯（多分頁） | 衝突偵測 + 提示 |

### 3. 測試報告產出 (Test Reporting)

- **結果格式**：Markdown 測試報告，含截圖證據
- **嚴重度分級**：

| 等級 | 定義 | 處理方式 |
|------|------|---------|
| 🔴 Critical | 跑版導致內容不可讀 | 必須修復才能上線 |
| 🟠 Major | 明顯視覺瑕疵但內容仍可讀 | 優先修復 |
| 🟡 Minor | 細微間距/對齊問題 | 排入後續修復 |
| 🟢 Cosmetic | 美觀建議 | 記錄但不阻擋 |

## 測試矩陣

### 切版測試矩陣

```
50 風格 × 3 內容長度（短/標準/長） × 5 斷點 = 750 組測試
```

**簡化策略**：
- 完整測試：每分類抽 2 個代表風格（共 10 個）× 3 長度 × 3 斷點 = 90 組
- 快速掃描：其餘 40 個風格 × 標準內容 × 桌面斷點 = 40 組
- **總計：130 組核心測試**

### API 穩定度測試矩陣

```
6 種 API 異常場景 × 3 種操作流程（開啟/儲存/導出）= 18 組測試
8 種 ContentEditable 異常場景 = 8 組測試
總計：26 組 API 測試
```

## 執行方式

### 切版測試流程

1. **準備測試內容**：建立短版（500 字）/ 標準版（3000 字）/ 長版（10000 字）測試報告
2. **自動化截圖**：使用瀏覽器開發者工具逐一載入 50 種風格
3. **人工審查**：依檢查清單逐項驗證
4. **問題記錄**：發現問題時截圖 + 記錄重現步驟 + 標記嚴重度
5. **產出報告**：彙整為 `_docs/qa-report-{date}.md`

### API 穩定度測試流程

1. **模擬異常環境**：逐一觸發各異常場景
2. **驗證防呆反應**：確認錯誤提示友善且操作可恢復
3. **邊界壓力測試**：連續觸發異常確認不崩潰
4. **記錄結果**：依測試矩陣填寫通過/失敗

## 品質檢查清單

### 切版測試

- [ ] 萬字長文在所有 50 種風格下不跑版
- [ ] 巨大圖表不超出容器邊界
- [ ] 超長標題正確斷行或截斷
- [ ] 巢狀清單縮排正確
- [ ] 密集表格具備水平捲動能力
- [ ] 手機端（375px）佈局不崩壊
- [ ] 導出 PDF 的分頁切割合理

### API 穩定度測試

- [ ] 拒絕授權後操作仍可繼續（降級模式）
- [ ] 磁碟空間不足提示明確
- [ ] 檔案鎖定時不靜默失敗
- [ ] 不支援瀏覽器的降級方案正常運作
- [ ] 貼上惡意內容已被過濾
- [ ] 超大內容的效能在可接受範圍
