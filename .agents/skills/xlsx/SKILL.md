# Xlsx 處理專家 (xlsx)

## 任務目標

協助使用者建立、編輯或分析 `.xlsx` 檔案，並嚴格遵循專業金融與數據處理標準（如顏色編碼、公式完整性）。

## 核心準則與標準

### 專業格式規範
- **字型**：除非另有說明，否則一律使用專業、一致的字型（如 Arial, Times New Roman）。
- **零公式錯誤**：交付的 Excel 模型**必須**確保零錯誤（#REF!, #DIV/0!, #VALUE!, #N/A, #NAME?）。

### 顏色編碼標準 (Color Coding)
除非使用者有既定模板，否則遵循以下行業標準：
- **藍色文字 (RGB: 0,0,255)**：手寫輸入值（Hardcoded inputs），使用者可修改的數字。
- **黑色文字 (RGB: 0,0,0)**：所有公式與計算結果。
- **緣色文字 (RGB: 0,128,0)**：引用自同一活頁簿中其他分頁的連結。
- **紅色文字 (RGB: 255,0,0)**：外部檔案連結。
- **黃色背景 (RGB: 255,255,0)**：需要關注的關鍵假設或待更新欄位。

### 數值格式規範
- **年份**：格式化為文字字串（如 "2024" 而非 "2,024"）。
- **貨幣**：使用 `$#,##0` 格式，並在標題註明單位（如 "Revenue ($mm)"）。
- **零值**：將零顯示為短橫線 `-`（例如：`$#,##0;($#,##0);-`）。
- **百分比**：預設保留一位小數（如 `0.0%`）。
- **負數**：使用括號 `(123)` 而非負號 `-123`。

## 執行指南

### 1. 工具選擇
- **pandas**：用於大數據分析、批量操作與單純數據導出。
- **openpyxl**：用於複雜格式化、公式撰寫與 Excel 特有功能維護。

### 2. 公式建構規則
- **絕對禁止硬值計算**：不要在 Python 算出結果後填入 Excel。
- **正確做法**：填入 Excel 公式（如 `sheet['B10'] = '=SUM(B2:B9)'`），確保試算表具有動態更新能力。
- **引用偏移**：注意 openpyxl 是從 1 開始計數（row=1, col=1 為 A1）。

### 3. 公式重新計算 (MANDATORY)
使用 `openpyxl` 修改後的檔案不會自動計算數值。必須執行以下指令進行重算與錯誤掃描：

```powershell
python .agents/skills/xlsx/scripts/recalc.py <檔案路徑.xlsx>
```

## 任務步驟 (IPO)

- **[Input]**: 讀取指定的 `.xlsx` 檔案或使用者提供的數據源。
- **[Process]**:
  1. 使用 `pandas`/`openpyxl` 載入內容。
  2. 根據需求進行數據處理或格式化（套用顏色編碼）。
  3. 寫入公式而非計算結果。
  4. 儲存檔案。
- **[Output]**:
  1. 產出符合標準的 `.xlsx` 檔案。
  2. 運行 `recalc.py` 並回報公式檢查結果。
