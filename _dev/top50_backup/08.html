⚡ TL;DR — 重點摘要
  • terra-sdk 是高效能的全端資料同步框架，支援離線優先架構
  • 內建衝突解決引擎，零設定即可實現多裝置資料一致性

  • 支援 TypeScript、React、Vue、Svelte 等主流前端框架

概覽

Terra SDK 是專為現代 Web 應用設計的離線優先資料同步框架。它在客戶端維護一份完整的資料庫副本，確保應用在任何網路環境下都能即時回應使用者操作。當連線恢復時，內建的 CRDT 衝突解決引擎會自動合併離線期間的所有變更。

💡

為什麼選擇 Terra？ 傳統的 REST/GraphQL 架構在離線場景下會完全癱瘓。Terra 透過 Event Sourcing + CRDT 的混合架構，讓你的應用天生具備離線能力，無需任何額外程式碼。

快速開始

安裝

bash 
# 使用 npm
npm install @terra/sdk

# 使用 pnpm（推薦）
pnpm add @terra/sdk

初始化

typescript 
import { Terra } from '@terra/sdk';

// 建立 Terra 實例
const db = new Terra({
  name: 'my-app',
  endpoint: 'wss://sync.terra.dev',
  auth: { token: process.env.TERRA_TOKEN }
});

// 定義資料模型
const TaskSchema = db.schema('tasks', {
  title: 'string',
  completed: 'boolean',
  assignee: 'string?',
  createdAt: 'datetime'
});

API 參考

以下是核心 API 方法的完整列表。所有方法皆支援 TypeScript 型別推斷。

方法參數回傳值說明
create()Partial<T>Promise<T>建立新記錄
findOne()Query<T>Promise<T | null>查詢單筆記錄
findMany()Query<T>Promise<T[]>查詢多筆記錄
update()id, Partial<T>Promise<T>更新記錄
subscribe()Query<T>, cbUnsubscribe即時訂閱變更

設定選項

⚠️

注意： conflictStrategy 設定一旦初始化後即無法變更。請在專案初期謹慎選擇衝突解決策略。

typescript 
const config: TerraConfig = {
  // 同步模式：'realtime' | 'batch' | 'manual'
  syncMode: 'realtime',

  // 衝突策略：'lww' | 'crdt' | 'custom'
  conflictStrategy: 'crdt',

  // 離線佇列最大容量（預設 1000）
  offlineQueueSize: 5000,

  // 自動重連間隔（毫秒）
  reconnectInterval: 3000
};

使用範例

即時待辦清單

typescript 
// 訂閱即時更新
const unsubscribe = TaskSchema.subscribe(
  { completed: false },
  (tasks) => {
    console.log(`待完成任務: ${tasks.length}`);
    renderTaskList(tasks);
  }
);

// 新增任務（離線時自動進入佇列）
await TaskSchema.create({
  title: 'Review PR #42',
  completed: false,
  assignee: 'alice'
});
