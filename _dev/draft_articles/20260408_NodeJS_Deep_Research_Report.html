3.1 核心架構與底層原理

核心高併發解法：
非同步與非阻塞模型

  • 依附於 V8 引擎與 libuv（統管 Event Loop 與 Thread Pool）。
  • 單執行緒 (Single-Threaded) 免除死鎖 (Deadlocks) 或 Race Conditions。
  • I/O 委託 Kernel 或 Thread Pool，回呼循序送回 V8 處理。
  • ESM 在 2026 生態下終於打破高牆，Tree-shaking 成為伺服器標配。
"

從底層的 C++ 綁定、V8 垃圾回收機制，到上層高度結構化的 NestJS 框架、gRPC 微服務通訊，其體系已經無懈可擊。面對運算密集任務，叢集與Worker Threads 已能應對。

3.2 效能 & 資安

效能瓶頸痛點

Memory Leak 多由未清除的閉包引起。業界標準：透過 `--inspect` 比較 Heap Snapshots，配合 Clinic.js 繪製火焰圖檢測 CPU 阻塞點。

NPM 供應鏈防禦

全面阻斷 `postinstall`、嚴格鎖定 `package-lock.json` Hash 值，並引入 Snyk 等 SCA 掃描，作為現代 CI/CD 必要配置。
