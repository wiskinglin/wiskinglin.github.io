# KLIO COLOR 全域配色測試與對比度驗證報告

## 測試目標與背景
針對 `tools/color.html` 內的 **所有 30 款配色主題** 進行全面性的 UI 渲染對比度深度檢測。確保套用任何一組動漫、專業或生活情緒的色票時，動態預覽小工具的文字與背景對比度均能達標。

**對比度測試標準 (WCAG 標準)：**
- `C1`（主要背景）對 `C5`（主要文字） > 4.5
- `C2`（表面背景）對 `C5`（主要文字） > 4.5

---

## 測試結果盤點 (共 12 組未達標)
透過撰寫檢測腳本，模擬了 30 款配色的表面與背景交疊運算，挑出了以下 **12 款對比度嚴重不及格**（會造成黑底黑字、白底白字或刺眼模糊）的配色：

1. **葬送的芙莉蓮** (文字不夠深)
2. **膽大黨** (C1光害、粉紅字反差不足)
3. **福音戰士 EVA** (C2 深背景配紫字)
4. **鬼滅之刃 炭治郎** (C2 深棕背景配黑字)
5. **鏈鋸人** (C2 深綠灰背景配黑字)
6. **創業活力 (Startup)** (糖果色導致文字過亮)
7. **教育學習 (Edu)** (C2 暖橘底色與文字衝突)
8. **網路行銷 (Marketing)** (C2 過亮/文字無反差)
9. **復古 80s (Retro)** (背景過飽和，字不夠深)
10. **春季櫻花 (Sakura)** (全粉彩色系，無暗色文字)
11. **甜點屋 (Pastel)** (全粉彩色系，無暗色文字)
12. **沙漠旅行 (Desert)** (C1, C2 顏色過重偏深)

## 優化與調色修復策略
我已經對這 12 組破壞 UI 邏輯的色盤進行了「全域重新校準」，遵守：
**`C1 (極淺底) -> C2 (淺底) -> C3 (鮮豔點綴) -> C4 (深色次標) -> C5 (極深主標)`** 的光學變化邏輯。

**精選校正範例：**
- **春季櫻花 (Sakura)**：
  _原本_全是輕柔的粉藍與粉紅色，導致文字完全融入背景消失。
  _修復後_：保留粉色櫻花背景 (`#FFF0F5`, `#FFE3EE`)，將文字改為具有春季感但足夠深的**櫻桃暗紅** (`#4D2C3B`) 及**深湖水藍** (`#4A7694`)。
- **福音戰士 EVA**：
  _原本_C2 為重工業黑 (`#1E1E1E`) 搭配紫色文字 (`#7030A0`)。
  _修復後_：將背景轉為冷白色調的 NERV 實驗室感 (`#FAFAFA`, `#F0F0F0`)，並將初號機的**紫綠色**保留於文字與高光元件上。
